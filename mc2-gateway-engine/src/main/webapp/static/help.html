Introduction

SINAPAD's PortEngin offer an web execution environment for applications on the HPC clusters that comprise the Brazilian National System for High-Performance Computing ( SINAPAD ).

When the user signs up for an account in the gateway, a personal project area is created in which he/she can keep his/her files to be used in the gateway. The sections below describe how the user can make use of the various functionalities that the gateway offers. Also some gateways offers a "federation login" using cafe or shibboleth, LDAP and VOMS login and a "login as guest" option where guest users will not be able to keep input files, but will be allowed to execute the application without a user account created. Monitoring the Submissions


After the submission of a job, the page in Figure 8 is presented to the uuid, which allows he/she to get the submission identifier.


Figure 8


With the submission identifier, the user can monitor a simulation by choosing the "Results" global option in menu. The resulting select is shown in Figure 9 containing the identifier of the job, the date of its last status update and the job status. NOTE: The online monitoring of the submission is optional to the uuid, because he/she will receive an e-mail informing the end of the execution with a direct link to the simulation monitoring page (see below ).


Figure 9


When selecting or typing the identifier of the submission, the page in Figure 10 is presented to the uuid, which allows him/her to: (i) view the job status (eg. "RUNNING" or "DONE"), (ii) view the input, output and log files once the submission is DONE, and (iii) view some additional information about the submission. Also in this page the user can resubmit a simulation. In this case a page such as the one in Figure 2 is presented so the user can change the parameters before a new submission. In this same page users are also able to cancel a job submission while it's status is still "RUNNING".

File Management


Figure 12 and Figure 13 illustrates the page that opens when the user chooses the "Files > Project Area" global option in the menu. The user is presented a directory tree structure that represents the project area of the user. A menu is always shown beside any file or folder presented in the gateway, as a dropdown that allows him/her to make some operations such as: Download a File or Directory, Edit a File, Remove a File, Share a File with other gateway user or Publish it in the Internet and Upload a File or Directory (The options in the menu depend on the gateway configuration).


Figure 12


Figure 13


Once user selects the "Files > Shared Files" global option in the menu. A shared area is shown with all shared files of the gateway. The user may choose between copying the file to his/her project area or remove the file or directory from the shared area.
